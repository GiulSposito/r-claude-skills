# Topic Modeling Guide

Complete reference for discovering latent topics in text using Latent Dirichlet Allocation (LDA) and related methods.

## Overview

Topic modeling automatically discovers abstract "topics" in a collection of documents. Each topic is a distribution over words, and each document is a mixture of topics.

**Key Concept**: LDA assumes:
- Each document contains multiple topics
- Each topic is characterized by a distribution of words
- Words in a document are generated by selecting a topic, then selecting a word from that topic

## Latent Dirichlet Allocation (LDA)

### Basic LDA Workflow

```r
library(topicmodels)
library(tidytext)
library(tidyverse)

# 1. Create document-term matrix
dtm <- tidy_text |>
  count(document, word) |>
  cast_dtm(document, word, n)

# 2. Fit LDA model
lda_model <- LDA(dtm, k = 5, control = list(seed = 123))

# 3. Extract topic-word probabilities (beta)
topics <- tidy(lda_model, matrix = "beta")

# 4. Extract document-topic probabilities (gamma)
doc_topics <- tidy(lda_model, matrix = "gamma")
```

### Understanding Beta (Topic-Word Distribution)

Beta represents P(word | topic) - the probability of each word appearing in each topic.

```r
# Extract beta
topics <- tidy(lda_model, matrix = "beta")

# Structure
head(topics)
#   topic term        beta
#      1  data   0.0123456
#      1  model  0.0234567
#      1  time   0.0145678
#      2  data   0.0098765
#      2  model  0.0187654

# Top 10 terms per topic
top_terms <- topics |>
  group_by(topic) |>
  slice_max(beta, n = 10) |>
  ungroup() |>
  arrange(topic, -beta)

# Visualize
top_terms |>
  mutate(term = reorder_within(term, beta, topic)) |>
  ggplot(aes(beta, term, fill = factor(topic))) +
  geom_col(show.legend = FALSE) +
  facet_wrap(~topic, scales = "free") +
  scale_y_reordered() +
  labs(title = "Top 10 Terms per Topic", x = "Beta (probability)", y = NULL)
```

### Understanding Gamma (Document-Topic Distribution)

Gamma represents P(topic | document) - the proportion of each topic in each document.

```r
# Extract gamma
doc_topics <- tidy(lda_model, matrix = "gamma")

# Structure
head(doc_topics)
#   document topic    gamma
#   doc1        1  0.234567
#   doc1        2  0.123456
#   doc1        3  0.456789
#   doc1        4  0.098765
#   doc1        5  0.086423

# Topic prevalence across documents
doc_topics |>
  group_by(topic) |>
  summarise(mean_gamma = mean(gamma)) |>
  ggplot(aes(factor(topic), mean_gamma)) +
  geom_col() +
  labs(title = "Average Topic Prevalence", x = "Topic", y = "Mean Gamma")

# Assign each document to dominant topic
doc_classification <- doc_topics |>
  group_by(document) |>
  slice_max(gamma, n = 1) |>
  ungroup()

# Topic distribution
doc_classification |>
  count(topic) |>
  ggplot(aes(factor(topic), n)) +
  geom_col() +
  labs(title = "Documents per Topic", x = "Topic", y = "Count")
```

## Choosing Number of Topics (k)

### Method 1: Perplexity

Lower perplexity indicates better fit (but watch for overfitting).

```r
library(purrr)

# Fit models with different k
models <- tibble(k = 2:10) |>
  mutate(
    lda_model = map(k, ~LDA(dtm, k = .x, control = list(seed = 123))),
    perplexity = map_dbl(lda_model, perplexity, newdata = dtm)
  )

# Plot perplexity
ggplot(models, aes(k, perplexity)) +
  geom_line() +
  geom_point() +
  labs(title = "Model Perplexity by Number of Topics",
       x = "Number of Topics (k)",
       y = "Perplexity (lower is better)")

# Look for "elbow"
```

### Method 2: Topic Coherence

Higher coherence indicates more interpretable topics.

```r
# Simplified coherence metric (based on term co-occurrence)
topic_coherence <- function(lda_model, dtm, top_n = 10) {
  topics <- tidy(lda_model, matrix = "beta") |>
    group_by(topic) |>
    slice_max(beta, n = top_n)

  # Calculate pairwise PMI (simplified)
  # Full implementation would use external corpus
  topics |>
    group_by(topic) |>
    summarise(coherence = mean(beta))
}

# Compare coherence across k
models <- models |>
  mutate(coherence = map_dbl(lda_model, topic_coherence, dtm = dtm))

ggplot(models, aes(k, coherence)) +
  geom_line() +
  geom_point() +
  labs(title = "Topic Coherence by k")
```

### Method 3: Manual Inspection

Often the best method - fit several k, inspect top terms, choose most interpretable.

```r
# Function to print top terms
print_top_terms <- function(lda_model, n = 10) {
  tidy(lda_model, matrix = "beta") |>
    group_by(topic) |>
    slice_max(beta, n = n) |>
    summarise(terms = paste(term, collapse = ", "))
}

# Compare different k
walk(2:8, ~{
  cat("\n=== k =", .x, "===\n")
  lda <- LDA(dtm, k = .x, control = list(seed = 123))
  print(print_top_terms(lda))
})
```

### Guidelines for Choosing k

| Corpus Size | Typical k Range |
|-------------|-----------------|
| < 100 documents | 2-5 topics |
| 100-1,000 documents | 5-20 topics |
| 1,000-10,000 documents | 10-50 topics |
| > 10,000 documents | 50-200 topics |

**General advice**: Start small (k=5), inspect topics, adjust based on interpretability.

## Preprocessing for Topic Modeling

### Standard Preprocessing Pipeline

```r
# Comprehensive preprocessing
processed_text <- raw_text |>
  # Tokenize
  unnest_tokens(word, text) |>
  # Remove stop words
  anti_join(stop_words, by = "word") |>
  # Remove numbers
  filter(!str_detect(word, "\\d+")) |>
  # Remove short words
  filter(nchar(word) >= 3) |>
  # Remove very rare words (appear in < 2 documents)
  group_by(word) |>
  filter(n_distinct(document) >= 2) |>
  ungroup() |>
  # Remove very common words (appear in > 90% of documents)
  add_count(word, name = "word_total") |>
  mutate(n_docs = n_distinct(document)) |>
  filter(word_total / n_docs < 0.9)
```

### Stemming for Topic Models

```r
library(SnowballC)

# Stem words before creating DTM
tidy_text |>
  mutate(stem = wordStem(word, language = "english")) |>
  count(document, stem) |>
  cast_dtm(document, stem, n)
```

### TF-IDF Weighting (Alternative)

```r
# Weight by TF-IDF before LDA
dtm_tfidf <- tidy_text |>
```

## Interpreting Topics

### Labeling Topics

```r
# Extract top terms
topic_labels <- tidy(lda_model, matrix = "beta") |>
  group_by(topic) |>
  slice_max(beta, n = 10) |>
  summarise(top_terms = paste(term, collapse = ", "))

# Manually assign labels
topic_labels <- topic_labels |>
  mutate(
    label = case_when(
      topic == 1 ~ "Data Analysis",
      topic == 2 ~ "Machine Learning",
      topic == 3 ~ "Statistics",
      topic == 4 ~ "Visualization",
      topic == 5 ~ "Programming"
    )
  )

# Use labels in visualizations
doc_topics |>
  left_join(topic_labels, by = "topic") |>
  ggplot(aes(factor(topic), gamma, fill = label)) +
  geom_boxplot()
```

### Topic Distinctiveness

```r
# Compare topics using beta divergence
# Terms that distinguish one topic from others
topic_diff <- topics |>
  pivot_wider(names_from = topic, values_from = beta, names_prefix = "topic_") |>
  mutate(
    log_ratio_1_2 = log2(topic_1 / topic_2),
    log_ratio_1_3 = log2(topic_1 / topic_3)
  )

# Words most distinctive to topic 1 vs topic 2
topic_diff |>
  slice_max(abs(log_ratio_1_2), n = 20) |>
  ggplot(aes(log_ratio_1_2, reorder(term, log_ratio_1_2))) +
  geom_col() +
  labs(title = "Words Most Distinctive Between Topics 1 & 2",
       x = "Log2 Ratio (Topic 1 / Topic 2)")
```

## Advanced Techniques

### Structural Topic Models (STM)

Incorporates document metadata into topic model.

```r
library(stm)

# Convert to STM format
stm_data <- tidy_text |>
  count(document, word) |>
  cast_sparse(document, word, n)

# Fit STM with covariates
stm_model <- stm(
  documents = stm_data,
  K = 10,  # Number of topics
  prevalence = ~year + category,  # Document-level covariates
  data = metadata,
  max.em.its = 75,
  seed = 123
)

# Examine effect of covariates on topic prevalence
effect <- estimateEffect(1:10 ~ year + category, stm_model, metadata)
summary(effect, topics = 1)
```

### Dynamic Topic Models

Topics evolve over time.

```r
# Requires external package like ldatuning or BTM
# Split corpus by time period
time_periods <- tidy_text |>
  mutate(period = cut(date, breaks = "year"))

# Fit separate LDA per period
period_models <- time_periods |>
  nest(data = -period) |>
  mutate(
    dtm = map(data, ~{
      .x |> count(document, word) |> cast_dtm(document, word, n)
    }),
    lda = map(dtm, ~LDA(.x, k = 5, control = list(seed = 123)))
  )

# Track topic evolution
period_topics <- period_models |>
  mutate(topics = map(lda, tidy, matrix = "beta")) |>
  unnest(topics)

# Visualize topic trends
period_topics |>
  group_by(period, topic) |>
  slice_max(beta, n = 5) |>
  ggplot(aes(period, beta, color = term, group = term)) +
  geom_line() +
  facet_wrap(~topic)
```

## Best Practices

### Preprocessing

✅ **Always** remove stop words (critical for topic models)
✅ **Always** remove very rare words (appear in < 2-3 documents)
✅ **Always** remove very common words (appear in > 50-90% of documents)
✅ **Consider** stemming (reduces vocabulary, can improve coherence)
✅ **Consider** removing numbers unless domain-specific (e.g., years)
✅ **Consider** bigrams for phrases ("machine learning", "new york")

### Model Selection

✅ **Try** multiple values of k (3, 5, 10, 20)
✅ **Use** perplexity as guide, but prioritize interpretability
✅ **Manually inspect** top terms for each k
✅ **Look for** topics that are coherent, distinct, and meaningful
✅ **Avoid** k too small (topics too general) or too large (topics too specific)

### Interpretation

✅ **Always** inspect top 10-20 terms per topic
✅ **Always** read representative documents for each topic
✅ **Always** assign meaningful labels to topics
✅ **Consider** topic correlations and co-occurrence
✅ **Validate** with domain experts if possible

### Reporting

✅ **Report** chosen k and selection process
✅ **Report** preprocessing steps
✅ **Report** top terms per topic (10-15)
✅ **Report** topic labels and interpretation
✅ **Show** example documents per topic
✅ **Show** topic prevalence (how common is each topic?)

## Common Pitfalls

❌ **Not removing stop words** → Topics dominated by "the", "and", "is"
✅ Always use `anti_join(stop_words)`

❌ **Too many rare words** → Noisy, uninterpretable topics
✅ Filter words appearing in < 2-3 documents

❌ **Choosing k arbitrarily** → Topics too general or too specific
✅ Try multiple k, inspect manually

❌ **Not inspecting results** → Meaningless topic labels
✅ Read top terms and example documents

❌ **Treating topics as "real"** → Topics are model artifacts
✅ Validate interpretations with domain knowledge

❌ **Ignoring preprocessing** → Garbage in, garbage out
✅ Careful preprocessing is critical

## Quick Reference

| Task | Code Pattern |
|------|--------------|
| Create DTM | `count(doc, word) |> cast_dtm(doc, word, n)` |
| Fit LDA | `LDA(dtm, k = 5, control = list(seed = 123))` |
| Extract topics (beta) | `tidy(lda, matrix = "beta")` |
| Extract doc-topics (gamma) | `tidy(lda, matrix = "gamma")` |
| Top terms per topic | `group_by(topic) |> slice_max(beta, n = 10)` |
| Dominant topic per doc | `group_by(document) |> slice_max(gamma, n = 1)` |
| Perplexity | `perplexity(lda, newdata = dtm)` |
| Log-likelihood | `logLik(lda)` |

## Example Workflow

```r
# Complete LDA workflow
library(topicmodels)
library(tidytext)
library(tidyverse)

# 1. Preprocess
processed <- raw_data |>
  unnest_tokens(word, text) |>
  anti_join(stop_words) |>
  filter(!str_detect(word, "\\d+")) |>
  add_count(word) |>
  filter(n >= 5) |>
  select(-n)

# 2. Create DTM
dtm <- processed |>
  count(document, word) |>
  cast_dtm(document, word, n)

# 3. Choose k
perplexity_df <- tibble(k = c(3, 5, 8, 10, 15, 20)) |>
  mutate(
    model = map(k, ~LDA(dtm, k = .x, control = list(seed = 123))),
    perp = map_dbl(model, perplexity, newdata = dtm)
  )

# 4. Fit final model
final_k <- 8
lda_final <- LDA(dtm, k = final_k, control = list(seed = 123))

# 5. Extract and label topics
topics <- tidy(lda_final, matrix = "beta")
doc_topics <- tidy(lda_final, matrix = "gamma")

topic_labels <- topics |>
  group_by(topic) |>
  slice_max(beta, n = 10) |>
  summarise(terms = paste(term, collapse = ", "))

# 6. Visualize
topics |>
  group_by(topic) |>
  slice_max(beta, n = 10) |>
  ggplot(aes(beta, reorder_within(term, beta, topic))) +
  geom_col() +
  facet_wrap(~topic, scales = "free") +
  scale_y_reordered()
```
